Interface DefinerAPI

An API object for defining modules. It is passed to a user-supplied definer callback that is default-exported from a javascript module file read by the framework.

after

after: ruleNominator
  • Defined in ts/front/module-builder.ts:69

Specifies rules after which this rule must be processed. It is a way to create dependency edges between rules themselves, rather than between artifacts. It is sometimes necessary when we want to depend on another rule's autotargets but we can't enumerate those in advance.

This function can only be used in a rule definer, but it is available in a module definer API object for convenience, so it can be destructured once per module.

also

also: ruleNominator
  • Defined in ts/front/module-builder.ts:91

Specify also-rules for the rule being defined. It specifies that when this rule is required, then (an)other rule(s) are automatically also required, but it does not specify relative order of processing. The order is dictated by regular dependencies which may force the other rules to be processed before or after the current rule, or allow them to be processed in parallel.

This function can only be used in a rule definer, but it is available in a module definer API object for convenience, so it can be destructured once per module.

always

always: flagSetter
  • Defined in ts/front/module-builder.ts:81

Mark the rule currently being defined as an always-rule. This replaces the normal up-to-date check for the rule's outputs with false. As a result, the rule's recipe is always invoked if any of its outputs is required to build the requested goal.

Resolve artifact-resolvables to Dependency instances and designate them as dependencies for the rule being defined.

include

include: includeNominator
  • Defined in ts/front/module-builder.ts:39

Include other modules by referring to directories containing them, relative to this module's directory. This is how you recursively include all your project's modules from the root module. There is no automatic scanning for .zrup.mjs files in subdirectories.
